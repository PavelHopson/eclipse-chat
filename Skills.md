# Skills.md — навыки агента

## Проверка: папки skills в репозитории НЕТ

Проверено 2026-05-22 — в `eclipse-chat/` каталога `skills/` или
`.claude/skills/` нет.

**Skills — это механизм Claude Code, не часть репозитория.** Они живут
глобально у пользователя: `~/.claude/skills/` (на машине Pavel'я —
`C:\Users\garaa\.claude\skills`). Доступны во всех проектах.

Если для Eclipse Chat понадобятся **проектные** скиллы (специфичные
рабочие процессы именно этого репо) — их можно завести в
`.claude/skills/` внутри репозитория, и они поедут вместе с кодом.
Пока такой потребности нет.

## Скиллы, релевантные для работы над Eclipse Chat

Вызываются как `/<имя>` либо подхватываются автоматически.

### Дизайн (основной фокус сейчас)
- **redesign-skill** — аудит generic/AI-паттернов, апгрейд до premium.
  Профильный инструмент текущего визуального передела.
- **frontend-design**, **ui-ux-pro-max**, **design**, **design-system** —
  построение интерфейсов, токены, компоненты.
- Точечные: **color-system**, **typography-scale**, **motion-system**,
  **visual-hierarchy**, **spacing-system**, **dark-mode-design**,
  **accessibility-audit**.

### Инженерия / проверка
- **verify** — запустить приложение и убедиться, что правка работает.
- **run** — запустить/показать приложение.
- **security-review** — security-ревью изменений ветки.
- **review** — ревью pull request'а.
- **claude-api** — если трогаем AI-фичи (Anthropic SDK, prompt caching).

### Прочее
- **update-config** — настройки harness'а (хуки, permissions).
- **loop**, **schedule** — повторяющиеся / отложенные задачи.

> Полный список доступных скиллов Claude Code показывает в начале
> сессии. Здесь — только профильные для этого проекта.
